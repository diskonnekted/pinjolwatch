Anda adalah Senior Laravel Developer & System Architect. Tugas Anda adalah membuat aplikasi web full-stack bernama "PinjolWatch" menggunakan Laravel 11+, MySQL 8, Blade/Livewire, Tailwind CSS, Alpine.js, dan Leaflet.js. Aplikasi ini berfungsi sebagai wadah pengaduan masyarakat terkait ancaman ilegal debt collector pinjol. Fokus utama: privasi ketat, pemetaan sebaran per Kabupaten, dan statistik agregat.

🎯 TUJUAN APLIKASI
- Menerima laporan anonim/pseudonim dari masyarakat
- Menyediakan pelacakan status tiket pengaduan
- Menampilkan peta sebaran & statistik per Kabupaten (tanpa koordinat presisi demi privasi)
- Dashboard admin untuk moderasi, verifikasi, deduplikasi, dan ekspor laporan resmi
- Kepatuhan UU PDP No. 27/2022 (enkripsi, anonymization, data retention, consent)

🛠️ TECH STACK & LIBRARY
- Backend: Laravel 11+ (PHP 8.2+), MySQL 8
- Frontend: Blade Templates, Livewire 3, Tailwind CSS 3, Alpine.js
- Mapping: Leaflet.js + GeoJSON Kabupaten Indonesia (sumber: https://github.com/ans-4175/peta-indonesia atau similar)
- Auth & RBAC: Laravel Breeze + Spatie/laravel-permission
- Security: Laravel Encryption, Rate Limiting, reCAPTCHA v3, 2FA (via Laravel Fortify)
- Storage: Local/S3 (encrypted), Intervention Image (optional for watermark/redaction)
- Validation: Laravel Form Requests + Custom Rules

🗺️ SPESIFIKASI PEMETAAN & STATISTIK
- Hanya tampilkan level Kabupaten/Kota. TIDAK menyimpan koordinat GPS pelapor.
- Simpan `kabupaten_id` (referensi ke tabel `kabupaten` berisi kode BPS, nama, batas wilayah GeoJSON).
- Agregasi: jumlah laporan, jenis ancaman terbanyak, aplikasi pinjol paling sering dilaporkan per kabupaten.
- Tampilkan di peta sebagai:
  • Choropleth (warna intensitas berdasarkan jumlah laporan)
  • Tooltip berisi statistik ringkas
  • Filter: rentang waktu, status verifikasi, kategori ancaman
- Gunakan caching (Redis/DB) untuk query agregat agar performa ringan.

🔒 SPESIFIKASI KEAMANAN & PRIVASI
- Semua bukti upload dienkripsi at-rest. Key terpisah dari DB.
- Form pelaporan tidak wajib NIK/email. Gunakan token tiket acak untuk tracking.
- Anonymizer otomatis: blur/redact wajah, NIK, nama lengkap pada bukti sebelum disimpan/tampil.
- Audit log mencatat setiap aksi admin (view, edit, export, delete) dengan IP, timestamp, user_id.
- Consent checkbox eksplisit sebelum submit: "Data boleh digunakan untuk verifikasi, diteruskan ke otoritas, atau hanya disimpan internal".
- Auto-delete/archive data setelah 24 bulan atau saat status `Selesai`.
- Rate limit: 3 submit/jam/IP. reCAPTCHA v3 wajib.
- Admin wajib 2FA & session timeout 15 menit.
